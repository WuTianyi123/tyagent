# tyagent 代码审查：快速实现遗留的技术债

审查日期：2026-05-02
审查范围：28 个源文件，3 个子代理全覆盖

## HIGH — 7 项

### 1. 伪造 User-Agent（agent.py L183, L381）
`"User-Agent": "KimiCLI/1.30.0"` 从 Kimi CLI 原样复制，向上游 LLM 谎报身份。

### 2. ~170 行 tool-call 循环重复（agent.py L174-343 vs L349-515）
chat() 流式和 _run_turn() 非流式包含近乎相同的 headers/payload/压缩重试/token追踪/tool执行逻辑，仅 ~15 行不同。

### 3. asyncio.sleep(999999) 哨兵（agent.py L580）
无 EventCollector 时用 999999 秒 sleep 模拟"永不完成"，浪费 asyncio 任务。应改用条件性加入 task 列表或 Event().wait()。

### 4. os._exit(0/75) 硬退出（lifecycle.py L142-143）
重启失败直接硬杀进程，跳过所有 finally/atexit。仅 WS 线程的 finally 有清理。

### 5. 共享 _default agent 竞态（gateway.py L237-243）
注释自认 race condition，标记为"CLI 没问题"但 Gateway 是多平台的。

### 6. 暂停 session 恢复丢消息（gateway.py L283-297）
检测 suspended session → 归档 → 通知 → return，用户发的消息未进 agent。

### 7. 模块 docstring 仍是旧 fallback（model_metadata.py L8）
FALLBACK_CONTEXT_LENGTH 和 L165 docstring 已改 256K，但模块顶 L8 仍写 128K。

## MEDIUM — 17 项

### 8. 硬编码参数全不可配（agent.py 多处）
HTTP timeout 120s、max_tokens 4096、temperature 0.7、shutdown timeout 5s。

### 9. 死赋值（agent.py L616→L630）
current_reply = None 在 L616 设置后被 L630 无条件覆盖。

### 10. callback 恢复重复 4 次（agent.py L644-676）
_tool_progress_callback 恢复在 try/AgentError/Exception/CancelledError 四处出现。

### 11. 重复 import 未清理（agent.py L23 vs L276/L433）
compress_context 模块级已 import，异常处理内又局部 import。

### 12. /new 后 agent 历史未刷新（gateway.py L434-462）
_ensure_session_agent 只在首次创建时加载 session.messages，/new 刷新 session 后旧 agent 留在缓存。

### 13. 并行字典管理状态（gateway.py L155-198）
__init__ 有 10+ 个独立字典，无 SessionContext 数据类，可能脱节。

### 14. API key 入 systemd unit（service_manager.py L80-84）
API key 写入 Environment= 字段，systemctl --user show 可见。

### 15. 消息永久丢失无日志（consumer.py L162）
编辑失败 fallback 中 except Exception: pass，消息静默消失。

### 16. 默认 provider/model 三处重复（config.py L32/L174/L205）
"anthropic/claude-sonnet-4" 和 "deepseek" 重复定义。

### 17. migrate.py doc 与实际不符（migrate.py L94 vs L126）
声称返回"number of messages imported"，实际 return 1（session 数）。

### 18. 重复 import uuid（session.py 5 处）
import uuid 在 5 个方法内各自内联 import。

### 19. 硬编码连接超时（gateway.py L171）
_AGENT_CACHE_MAX_SIZE=100 等魔术数字全不可配。

### 20. 脆弱 session_id 提取（gateway.py L334-335）
getattr(session.metadata, ...) 对 dict 无效，仅靠第二个 fallback 才工作。

### 21. 内置 platform 绕过 register_platform（gateway.py L139-147）
直接赋值 _PLATFORM_REGISTRY["feishu"] = FeishuAdapter，不用 register_platform()。

### 22. 重复 fallback 逻辑（consumer.py L233-246 vs L184-217）
CancelledError 处理的 ~30 行与 got_done 处理几乎相同。

### 23. feishu.py 四段重复（L1006-1020）
image/file/audio/media 四段仅 key 名不同，应用 dict 驱动的循环。

### 24. 文件扩展名映射两处定义（feishu.py L1284-1291 vs L1331-1347）
_upload_image 和 _upload_file 各自定义扩展名→类型映射，结构不同。

### 25. busyloop 等待（lifecycle.py L300-301）
for _ in range(20): await asyncio.sleep(0.5)，无 Event 或 callback。

### 26. os.kill 耦合（commands.py L141-142）
restart 命令用 os.kill(os.getpid(), SIGUSR1)，耦合 Unix 信号机制。
